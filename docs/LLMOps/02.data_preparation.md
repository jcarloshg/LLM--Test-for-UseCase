# Phase 2: Data Collection & Preparation

## Objective

- Organize data needed for LLM application. Including examples for prompts, evaluation datasets.
  > 100 high-quality examples are better option than 1,000 bad examples.

## Key Activities

- Collect representative input-output examples
- Clean and anonymize data (remove PII)
- Structure data in appropriate formats: JSON
- Version datasets for reproducibility **ToDo**
- Create evaluation datasets with ground truth labels **ToDo**

## Tools

- **Data manipulation**: Gemini to create 100 high-quality examples
- **Dataset versioning**: DVC (Data Version Control) **ToDo**
- **Data annotation**: Label Studio **ToDo**

## Outputs

- Few-shot examples for prompt engineering
- Evaluation dataset (train/val/test splits)
- Data quality report
- Versioned datasets with metadata **ToDo**

## Challenges

- **PII exposure:** Accidentally sending sensitive data to external APIs.
- **Label quality:** Poor human annotations lead to bad baselines.
- **Representation:** Test data doesn't match production distribution, causing performance gaps.
